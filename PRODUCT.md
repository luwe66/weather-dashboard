## 一、项目起源

在 Kiro 中与 AI 对话，决定做一个天气数据可视化大屏项目。

---

## 二、核心决策

### 数据来源

调研了以下免费天气 API：

| API | 免费额度 | 备注 |
|-----|---------|------|
| Open-Meteo | 完全免费，无需注册 | 无需 API Key |
| OpenWeatherMap | 1000次/天 | 需注册 |
| WeatherAPI | 100万次/月 | 需注册 |
| 和风天气 | 每月前5万次免费，按量计费 | 中文支持好，数据覆盖全国 |
| 彩云天气 | 有免费层 | 分钟级降水预报准确 |

**决策：选用和风天气**
- 中文支持好
- 数据覆盖国内城市完整
- 每月5万次免费额度对大屏场景足够
- 文档完善，开发体验好

---

### 技术栈

对比了三种方案：

- **原生 HTML/JS**：门槛低，但大项目难维护
- **React**：生态大，全球最流行，稍复杂
- **Vue**：国内生态成熟，语法直观

**决策：Vue 3 + ECharts + Vite**
- 国内大屏项目 Vue 生态更成熟
- ECharts 是百度出品的图表库，中文文档完善，图表效果好
- 大屏 + ECharts 是非常成熟的技术组合
- 项目本身就需要用到 ECharts 图表

**用量数据存储：Firebase Realtime Database**
- 用量数据需要多端同步（本地、Vercel、同事浏览器看到同一份数据）
- localStorage 只能本地存储，无法跨设备同步
- Firebase 免费额度足够，无需搭建后端，直接用 REST API 读写

## 三、功能模块（v1.0 已实现）

### 核心模块

- [x] 实时天气：温度、天气状况、体感温度、湿度、风速风向、能见度
- [x] 未来 7 天预报（含温度区间可视化）
- [x] 24 小时温度折线图 + 降水概率柱状图（ECharts）
- [x] 日出日落弧形动画
- [x] 城市搜索切换
- [ ] 空气质量 AQI 仪表盘（和风天气新版 API 需要 JWT 认证，待处理）

### API 用量控制模块

背景：和风天气按量计费，每月前 5 万次免费，超出后收费。

- [x] **请求开关**：全局开启/关闭自动刷新，关闭后停止所有 API 请求
- [x] **自动关闭保护**：当月请求累计达到 30,000 次时，自动关闭开关并提示用户
- [x] **历史用量统计**：按月记录请求次数，ECharts 柱状图展示趋势，长期保留不清零
- [x] **当月用量进度条**：显示本月已用 / 50,000，颜色随用量变化（绿→黄→红）
- [x] **多端数据同步**：用量数据存储在 Firebase，所有人看到同一份统计

- 代码托管：GitHub
- 线上部署：Vercel（自动 CI/CD，push 代码自动更新）

## 七、设计工作流

**设计与开发协作方式**

采用 Figma ↔ 代码双向同步的工作流：

1. 通过 Kiro 将当前页面转换为 Figma 设计稿
2. 在 Figma 中进行视觉细节调整（间距、颜色、字体、布局等）
3. Kiro 读取修改后的 Figma 文件，同步更新代码

**好处**：
- 设计修改在 Figma 中直观操作，不需要直接改 CSS
- 改动有设计稿留档，方便回溯
- 设计和代码始终保持一致

记录开发过程中遇到的问题和最终解决方案，避免重复踩坑。

### 1. 和风天气 API 403 问题

**问题**：API 请求返回 403，页面无法加载数据。

**原因**：和风天气新版不再使用共享域名（`devapi.qweather.com`），每个账号有专属的 API Host。

**解决**：去控制台 → 设置 → 找到 API Host（格式如 `xxx.re.qweatherapi.com`），替换代码里的请求域名。

---

### 2. 空气质量 API 403 问题

**问题**：`/v7/air/now` 接口返回 403。

**原因**：和风天气新版空气质量 API 路径改为 `/airquality/v1/current/{lat}/{lon}`，且需要 JWT 认证，不再支持直接传 API Key。

**状态**：暂时跳过，空气质量模块隐藏，待后续实现 JWT 签名逻辑。

---

### 3. 日出日落时间格式问题

**问题**：天文 API 返回的日出日落格式是 `2026-05-10T05:06+08:00`，直接显示会带日期。

**解决**：用 `t.slice(t.indexOf('T') + 1, t.indexOf('T') + 6)` 截取时间部分。

---

### 4. API 用量多端不同步问题

**问题**：本地和 Vercel 上看到的用量数字不一样，因为 localStorage 是各浏览器独立存储的。

**解决**：改用 Firebase Realtime Database 存储用量数据，所有端读写同一份数据，实现多端同步。

---

### 5. Figma 捕获页面尺寸偏大问题

**问题**：Mac 使用 2K 分辨率时，捕获到 Figma 的页面尺寸是 2401x1224，而不是预期的 1920x1080。

**原因**：Mac 的 HiDPI 缩放导致实际渲染像素数偏大。

**解决**：
1. 系统设置 → 显示器 → 分辨率调整为 1080p
2. 浏览器点右上角全屏按钮进入全屏
3. 全屏状态下按 `Cmd + L` 聚焦地址栏，粘贴捕获 URL 并回车
4. 捕获完成后把分辨率改回来
